# Roadmap — world-class commentary arc (BV + BGFA)

Scoped arc to reach world-class content quality on two laws: **BV** (constitutional,
flagship, 233 articles) and **BGFA** (professional regulation, smallest corpus, 38
articles). The pair is deliberate: BV gives depth and citation density, BGFA gives
fast iteration feedback and a natural testbed for debugging the broken preparatory-
materials pipeline. If the prompt interventions generalise across two very different
legal domains, we have evidence they'll generalise to the other seven laws later.

Goal: **bring BV and BGFA to a quality bar where the automated audit shows
<5% citation mismatch on BSK, BBl is verified against real preparatory-materials
data, and random spot-checks hold up to expert review** — in that order, measurable
at every step.

## Guiding principles

1. **Measurement before intervention.** Every phase begins by establishing a baseline
   and ends by measuring the delta. No vibes-based quality claims.
2. **One variable at a time.** Each phase changes one thing. If phase 1 changes the
   prompt and phase 0 changed the model, phase 0's effect must be measured before
   phase 1 starts so we don't confound them.
3. **Small samples before full rollouts.** Never regenerate >10 articles without
   measuring on a sample first.
4. **Stop conditions per phase.** Every phase has an explicit condition under which
   we bail out and re-brainstorm rather than pushing through.
5. **Cost discipline.** The arc budget envelope is ~$930-$980. Phase-by-phase
   estimates in the table below. Full rollouts are their own decisions, not
   implicit in phase approvals.

## Phase 0 — infra hygiene

### Scope

Three changes, all preconditions for subsequent phases:

1. **Model bump:** `agents/anthropic_client.py` MODEL_MAP and the two digest
   scripts (`scripts/digest_botschaften.py`, `scripts/digest_commentary.py`)
   move from `claude-opus-4-20250514` / `claude-sonnet-4-20250514` to
   `claude-opus-4-6` / `claude-sonnet-4-6`. Free quality gain at same price
   point. Affects every subsequent API call in the project.

2. **Prompt caching:** `agents/anthropic_client.py` wraps the system prompt
   in a content block with `cache_control: {type: "ephemeral"}`. In the
   agentic loop (~8-12 turns per article), the ~15k-token static portion
   currently gets re-billed on every turn. With caching, it's billed once
   at ~1.25x and re-read at ~0.10x — net savings of roughly $2/article on
   Opus doctrine regeneration, or ~$540 across the 271-article rollout in
   phase 2. The single largest cost lever in the project.

3. **Smoke test:** 4 articles spanning BV and BGFA, article complexity
   bands, and a mix of tiers:
   - `BV Art. 8` (Gleichheit — tier 1, many citations)
   - `BV Art. 67` (Kulturkompetenz — medium complexity, ref data spot-checked)
   - `BGFA Art. 8` (professional regulation, average article)
   - `BGFA Art. 1` (purpose article, structurally different)

   For each: capture before-state (current doctrine.md, meta.yaml score,
   citation audit count), run `pipeline single {law} {n} --layer doctrine`,
   capture after-state including Anthropic API usage metrics with cache
   read/write counts, compute actual $/article.

### Success criteria

- MODEL_MAP bump works (API accepts `claude-opus-4-6` / `claude-sonnet-4-6`)
- `usage.cache_creation_input_tokens` and `usage.cache_read_input_tokens` are
  non-zero on turns ≥ 2 — confirms caching is active
- Smoke-test per-article doctrine cost comes in at ≤ $2.25 (the estimate)
- Smoke-test output quality is subjectively comparable to or better than the
  pre-regeneration baseline (no obvious regressions in structure, citation
  count, or Randziffer formatting)
- Existing test suite still passes (tests mock `run_agent` so they're
  indifferent to model IDs and caching)

### Stop conditions

- Wrong model ID → verify against Anthropic SDK docs, fix, retry
- Caching API version mismatch → commit model bump alone, defer caching
- Smoke-test quality regression → roll back model bump, investigate offline
- Smoke-test per-article cost >20% above estimate → halt, investigate before
  phase 1

### Deliverables

Commit A: model bump (MODEL_MAP + digest script defaults)
Commit B: prompt caching + cache-aware cost accounting
Commit C: smoke test results document capturing the four articles' metrics

Phase 0 is explicitly not a content-quality phase. It sets up the
infrastructure and gives us accurate per-article cost data to make phase 1
planning more concrete.

## Phase 1 — diagnosis + intervention (BSK mismatches)

The motivating number: 290 of 1237 BV BSK citations (23.5%) fail the audit.
But the audit only checks the top-level `authors` list in `bv_refs.json` —
`positions[].author` isn't checked, so there are false positives. We can't
trust the baseline until we fix the measurement.

Four sub-steps, each with a stop condition:

**1a — Fix audit accuracy.** Extend `_verify_bsk_citation` in
`scripts/verify_citations.py` to also match against `positions[].author` and
`controversies[].positions`. Introduce a three-state match result
(strict / loose / no-match). Re-run on BV and BGFA. Report before/after
distribution.

Stop condition: if the no-match count drops below 50, the BSK fabrication
problem is much smaller than we thought → pivot to a different quality gap
or skip to phase 2.

**1b — Diagnose a sample.** Pick 5 articles still showing no-match citations
after 1a. For each flagged citation, classify into TYPE A (real author, ref
data incomplete), TYPE B (wrong commentary attribution — source-discipline
confusion), or TYPE C (plausibly fabricated). Write diagnosis findings to
a companion spec doc.

The dominant type determines the intervention layer:
- TYPE A → ref data maintenance, no prompt change
- TYPE B → law-agent prompt tightened on source discipline
- TYPE C → both law-agent and evaluator prompts tightened

**1c — Intervene at the right layer.** One targeted edit. Not a rewrite.

**1d — Validate against a sample.** Regenerate 10 affected articles
(7 BV + 3 BGFA), re-run audit on the sample, run the evaluator, compare
mismatch rate and scores before/after. If both hold, proceed to phase 2.
If not, back to 1b.

Expected iteration count: 2-3 rounds before a prompt lands cleanly.

## Phase 2 — rollout (BV + BGFA)

Apply the validated phase 1 intervention to all 233 BV doctrine layers and
all 38 BGFA doctrine layers (271 total). Run the evaluator on each. Re-run
the citation audit on both laws. Confirm mismatch rate drops materially.

Manual triage anything still flagged after regeneration — these are the
hard cases worth human attention.

Out of scope for phase 2: other laws (OR, ZGB, etc.), other layers
(summary, caselaw), other citation types.

## Phase 3 — prep-data gap

Investigate why `scripts/preparatory_materials/bgfa.json` came out empty
on 2026-04-06. The file exists, was generated, but contains zero articles.
Diagnose which stage of `preparatory_materials_pipeline` is failing
(discovery, download, extraction, or digestion) and fix it.

Once the pipeline works, run it for BGFA (38 articles) then BV (233
articles). Verify the output has meaningful article coverage. Re-run the
citation audit: the 607 BV BBl citations and BGFA's BBl citations should
move from "unchecked" to verified/flagged, giving us genuine coverage on
legislative-history references.

## Phase 4 — beta launch polish

Independent of content work. Four small pieces:

1. **Analytics:** privacy-respecting (Plausible or similar), not Google
2. **Feedback channel:** at minimum a GitHub issue link in the footer, ideally
   a per-article "report an issue" affordance
3. **Methodology disclaimer:** explicit `/about` and `/methodology` sections
   clarifying AI generation, limitations, expected verification by users
4. **SEO + accessibility polish:** proper 404 page, JSON-LD structured data,
   sitemap validation, lighthouse audit

All API-free. Can happen in parallel with or after phases 1-3.

## Out of the arc entirely

These were on the salvage branch or in prior brainstorms but explicitly
excluded from this arc:

- Cantonal laws expansion (mission drift from federal scope)
- Deep multi-pass generation pipeline (speculative infrastructure)
- Editorial workflow / human-editor pairing (no users yet)
- Quality dashboard aggregation (premature — no stable metrics yet)
- Article tier classification in code (wrong shape, belongs in meta.yaml)
- Prompt "architecture" rewrite (the runaway failure mode we're avoiding)
- `fix_citations.py`-style auto-editing of content (destructive)

## Notes

The arc is staged so we can exit cleanly after any phase if priorities
change. Phase 0 produces accurate cost data; phase 1 produces a validated
prompt intervention OR proves the BSK problem is smaller than feared;
phase 2 produces cleaned content; phase 3 produces working prep data;
phase 4 produces a launchable site.

Each phase's spec lives in its own file under `docs/superpowers/specs/`
and its implementation plan under `docs/superpowers/plans/`. This roadmap
is the index, not the specs themselves.
